Intelligence Operations Guide (MDO C4ISR)

        
    

    


        
This platform is a Strategic OSINT Cyber Radar (MDO C4ISR) designed to detect Kinetic Warfare Precursors — indicators across Cyber, Physical, and Information domains that historically precede or accompany physical military action. All data is derived from open-source intelligence (OSINT). The system is calibrated against confirmed historical precedents: Russia→Ukraine (Feb 2022), China→Taiwan Strait (Aug 2022, Pelosi visit), and DPRK persistent operations against the Korean Peninsula.

        
Data refreshes automatically every 15 minutes. Use SYNC to force an immediate update. Sensor poll intervals vary by source (see Section 1).


        
1. Multi-Domain Sensor Architecture (9 Active Sensors)


        
Cyber Domain (Weight: 0.50) — Highest weighted domain. Nation-state cyber operations consistently precede kinetic action by 24–72 hours.

        
    
            
  • Cloudflare Radar API [Poll: 5 min]: L3 (volumetric) and L7 (application-layer) DDoS telemetry. Tracks per-origin traffic percentages globally, computes weighted spike ratios against a rolling 7-day baseline, detects botnet overlap across allied theaters, and identifies L7 vector escalation (HTTP flood shift). This is the highest-signal sensor for detecting coordinated state-directed attack campaigns. Data covers >20% of global internet traffic.
                
    Anti-inflation safeguards: Spike calculation is skipped if the 7-day baseline is not yet populated (startup guard). Spike ratio is capped at 25× to prevent statistical outliers from triggering false alerts. Origins with <1% absolute traffic share are excluded from the weighted average. Non-adversary states use a 3% baseline floor; adversary states (CN/RU/KP) use a 0.5% floor to preserve sensitivity to low-volume state operations.
            
    • 
            
  • ThreatFox (abuse.ch) API [Poll: 30 min]: Queries the ThreatFox threat intelligence feed for APT-tagged IoCs (Indicators of Compromise) published within the past 24 hours. Matches IoCs by APT tag or target country name. A hit indicates that known advanced persistent threat infrastructure is active and associated with the monitored theater. Authentication via THREATFOX_API_KEY (Auth-Key header, required since 2024 policy change).
    • 
            
  • RIPE Stat BGP Routing [Poll: 10 min]: Monitors BGP prefix announcements for the core theater via RIPE NCC. Triggers when announced prefix count drops by >15% relative to the prior poll, indicating mass route withdrawal — a known precursor to internet isolation maneuvers (ref: Russia 2022, Iran 2019).
    • 
        


        
Physical/Logistics Domain (Weight: 0.30) — Encompasses infrastructure, airspace, and kinetic pre-strike indicators.

        
    
            
  • IODA API (CAIDA) [Poll: 10 min]: Internet Outage Detection and Analysis. Tracks BGP-based internet reachability and active probing (ping) success rates for each monitored country. A simultaneous drop in both BGP routes and active probing indicates a significant infrastructure outage. This sensor correlates with BGP Routing for confirmation.
    • 
            
  • NASA EONET Wildfire API [Poll: 15 min]: Real-time open wildfire/thermal event data from NASA's Earth Observatory Natural Event Tracker. For each monitored theater, events within ±10° latitude/longitude of the theater's centroid coordinates are flagged. Wildfires near strategic military positions or logistics corridors may indicate kinetic strikes, artillery fires, or pre-conflict infrastructure destruction — particularly when correlated with GDELT tone collapse and cyber spikes. Note: This sensor replaced the original NASA FIRMS endpoint, which is unreachable from this network environment. EONET requires no API key. Coordinates are in GeoJSON [longitude, latitude] format.
    • 
            
  • OpenSky Network [Poll: 3 min]: Civilian airspace monitoring over predefined airport bounding boxes for each theater. Computes a 20-cycle rolling baseline of observed aircraft count. An ANOMALY is triggered when current count deviates by >40% from baseline; CLOSURE is triggered when count drops below 5% of baseline, indicating near-total civilian airspace suspension — historically observed 12–48 hours before military action begins. Weather suppression is applied when OpenWeatherMap reports severe conditions over the target.
    • 
            
  • PeeringDB / IXP Chokepoints [Poll: 60 min]: Static reference layer showing registered Internet Exchange Points (IXPs) and undersea cable landing stations critical to the monitored theater. Used as geographic context for interpreting BGP anomalies and physical infrastructure targeting. Displayed as cyan diamond markers on the map.
    • 
        


        
Information Domain (Weight: 0.20) — Propaganda and diplomatic tone analysis. Typically the earliest leading indicator, shifting 48–96 hours ahead of operations.

        
    
            
  • GDELT TimelineTone API [Poll: 15 min]: Queries the Global Database of Events, Language, and Tone for media coverage of the core theater. Computes a rolling 28-day average tone and compares against the configured alert threshold. A tone score below –12.0 constitutes an alert. Calibration reference: Russia→Ukraine onset showed –12 to –15; invasion-day readings reached –18 to –25. The –12.0 threshold provides early-warning sensitivity without excessive peacetime noise. Weather suppression is applied during severe weather conditions.
    • 
        


        
Noise Mitigation & Human-in-the-Loop (HITL) Control:

        
    
            
  • OpenWeatherMap [Poll: 30 min]: If severe weather (thunderstorms, blizzard, hurricane, etc.) is detected over the core theater, physical domain alerts (IODA BGP outages, OpenSky airspace anomalies) and information domain alerts (GDELT tone) are automatically marked SUPPRESSED and excluded from scoring. This prevents weather events from creating false escalations.
    • 
            
  • Analyst MUTE (HITL Override): Every sensor entry in the Evidence Panel shows a [ MUTE ] button. Clicking it immediately excludes that sensor's score contribution from the threat calculation and re-runs convergence. Muted sensors are persisted in browser local storage across sessions. Use this to suppress known false positives while preserving the full audit trail. When multiple sensors are muted simultaneously, the system correctly recalculates domain weights with remaining active sensors only.
    •

3. Scoring Engine & Convergence Algorithm

        
The WeightedConvergenceEngine evaluates a Raw Threat Score from all enabled, non-suppressed sensor signals, then applies a cross-domain convergence bonus. Maximum theoretical score exceeds 16 points.


        
Raw Score Components:

        
    
            
  • [Cyber] cf_spike_core:     Core theater weighted avg spike >2× (+1), >4× (+1), >6× (+1) — max +3
    • 
            
  • [Cyber] cf_adversary_strike: Adversary state (CN/RU/KP) direct strike ≥4× spike with >3% abs traffic → +2
    • 
            
  • [Cyber] cf_botnet_overlap:  Shared botnet overlap with allied theater >30% → +1
    • 
            
  • [Cyber] cf_vector_shift:    L7 app-layer avg spike >2.5× AND >1.5× greater than L3 avg → +1
    • 
            
  • [Cyber] cf_coordinated:     ≥2 strategic theaters simultaneously above 3× spike → +1
    • 
            
  • [Cyber] ripe_bgp:           BGP prefix withdrawal >15% → +1
    • 
            
  • [Cyber] threatfox:          APT-tagged IoC matched to theater in past 24h → +1
    • 
            
  • [Physical] ioda_bgp:        Confirmed BGP/probing outage (weather-filtered) → +1
    • 
            
  • [Physical] opensky:         Airspace ANOMALY (≥40% deviation from 20-cycle baseline) → +2
    • 
            
  • [Physical] opensky:         Airspace CLOSURE (<5% of baseline, near-total suspension) → +3
    • 
            
  • [Physical] nasa_firms:      NASA EONET wildfire within ±10° of theater centroid → +3
    • 
            
  • [Info]   gdelt:             GDELT media tone < –12.0 (28-day rolling avg, weather-filtered) → +1
    • 
        


        
Domain Convergence Bonus — Applied when anomalies simultaneously span multiple domains, indicating coordinated multi-domain operations (MDO) rather than isolated incidents:

        
    
            
  • DUAL CONVERGENCE: Any 2 of 3 domains (Cyber + Physical, or Cyber + Info, or Physical + Info) both active → +1 pt
    • 
            
  • FULL CONVERGENCE: All 3 domains (Cyber + Physical + Info) simultaneously active → +2 pts
    • 
        

        
A domain is considered "active" when its weighted score contribution exceeds the domain threshold. Full Convergence is the strongest signal for pre-kinetic operations — historical data shows this pattern in both the 2022 Ukraine invasion and the 2022 Taiwan Strait exercises.


        
Botnet Overlap Calibration: The 30% threshold reflects confirmed nation-state coordinated attack patterns (20–35% shared botnet infrastructure, per post-incident forensics of GRU/Sandworm operations). The previous 45% threshold matched large civilian botnet patterns (Mirai-class) and was too insensitive to state-directed attacks that use diverse, smaller botnets.


        
4. DEFCON Threat Levels

        
Final Score = Raw Score + Convergence Bonus. Hysteresis (1-cycle delay) is applied when downgrading levels to prevent flickering on borderline readings. Upgrades are immediate.

        
    
            
  • Level 1 — CRITICAL: Score ≥ 9 AND Hard Condition (Core Spike >5.0× AND confirmed BGP outage simultaneously). This compound condition requires both massive cyber surge and infrastructure collapse — the signature of a coordinated pre-invasion blackout operation. Imminent kinetic action probability is extremely high. Activate all contingency procedures.
    • 
            
  • Level 2 — SEVERE: Score ≥ 6. Multi-domain threat with direct adversary state attribution or severe infrastructure degradation. High probability of state-directed hostile action. Immediate command notification required.
    • 
            
  • Level 3 — HIGH: Score ≥ 4. Significant coordinated anomaly across at least two sensors. May indicate preparation phase, probing operations, or early escalation. Heightened monitoring warranted.
    • 
            
  • Level 4 — ELEVATED: Score ≥ 2. Early-stage indicators: media tone deterioration, isolated BGP events, or minor L7 vector shifts. Consistent with pre-crisis diplomatic deterioration or reconnaissance activity.
    • 
            
  • Level 5 — NORMAL: Score < 2. Background noise level. No significant anomalies across monitored domains.
    • 
        


        
5. Historical Calibration Reference

        
Thresholds in this system are calibrated against the following confirmed events:

        
    
            
  • Russia → Ukraine (Feb 24, 2022): BGP route withdrawals 10–15% in the 6 hours before invasion. Wiper malware (HermeticWiper) deployed via pre-positioned APT infrastructure 90 min before ground assault. GDELT tone reached –18 to –25 at invasion onset; –12 to –15 during the preceding weeks of military buildup. Civilian flights suspended 24h prior.
    • 
            
  • China → Taiwan Strait (Aug 2022, Pelosi visit): CN-origin DDoS traffic spiked 3–5× against TW government and media infrastructure. GDELT tone –13 to –17 during the exercise period. No BGP outage (deliberate — PRC signaling, not isolation). Botnet overlap with JP/US-targeted traffic confirmed in post-incident analysis.
    • 
            
  • DPRK persistent operations (ongoing): KP-origin traffic characteristically low in absolute percentage (0.1–0.5%) but highly persistent and targeted. The 0.5% adversary baseline floor preserves detection of these small-volume, high-significance operations that would be masked by a 3% floor.
    • 
        


        
6. Analyst Workflow & Best Practices

        
    
            
  • Cross-domain confirmation first: Do not escalate based on a single domain. A DDoS spike alone (Cyber only) is insufficient — civilian botnets routinely produce 3–5× spikes. Full Convergence (Cyber + Physical + Info) is the minimum standard for Level 2 escalation recommendations.
    • 
            
  • Use HITL MUTE for known false positives: If a sensor is producing noise from a confirmed non-hostile source (e.g., a major sporting event causing airspace anomalies, or a legitimate CDN maintenance window causing BGP shifts), mute it in the Evidence Panel. The mute state persists across page reloads via browser local storage.
    • 
            
  • SITREP for command reporting: Use the SITREP button to generate a formatted Situation Report suitable for command-level notification. The report includes domain breakdown, sensor evidence, and trend indicators.
    • 
            
  • Baseline awareness: Spike ratios are only computed when the 7-day rolling baseline has been populated. On first startup, the system requires approximately 7 days of sensor operation to establish accurate baselines. During this period, spike scores will be conservatively suppressed (has_baseline guard active). The Data Status panel shows baseline population progress per sensor.
    • 
            
  • GDELT tone interpretation: Tone of –12.0 marks the threshold for early military tension rhetoric in open media. Values of –15 to –18 indicate active crisis framing. Values below –18 historically correspond to imminent or ongoing kinetic action. A sudden 3–5 point tone drop within a single polling cycle (15 min) is more significant than a gradual decline to the same value.
