Intelligence Operations Guide (MDO C4ISR)

        
    

    


        
This platform is a Strategic OSINT Cyber Radar (MDO C4ISR) designed to detect Kinetic Warfare Precursors — indicators across Cyber, Physical, and Information domains that historically precede or accompany physical military action. All data is derived from open-source intelligence (OSINT). The system is calibrated against confirmed historical precedents: Russia→Ukraine (Feb 2022), China→Taiwan Strait (Aug 2022, Pelosi visit), and DPRK persistent operations against the Korean Peninsula.

        
Data refreshes automatically every 15 minutes. Use SYNC to force an immediate update. Sensor poll intervals vary by source (see Section 1). Phase 8 added 3 new sensors, a Derivative Engine, a Sequence Scorer, and four new HUD gauges.


        

            
PHASE 8 — NEW IN THIS VERSION

            
    
                
  • DDoS Acceleration Engine — 1st/2nd derivative of threat score. Ambush Pattern detection (2階微分 Z-Score > 2.0).
    • 
                
  • RSS Narrative Burst Sensor — Z-Score analysis of TASS / Xinhua / Mehr / RT / KCNA feeds against a 30-day baseline. Filters daily propaganda noise; alerts only on statistically significant tactical keyword spikes.
    • 
                
  • ISR Hotspot Sensor — OpenSky Network filtered by ISR aircraft characteristics (high-alt >9000 m, low-speed <160 m/s, military squawk/callsign) over 10 strategic chokepoints. Independent of civilian airspace monitoring.
    • 
                
  • AIS Maritime Sensor — AISHub public data for undersea cable chokepoints. Detects AIS Dark Gaps (≥1 h transmission blackout) and stationary non-commercial vessels.
    • 
                
  • Sequence Scorer — Validates the logical escalation chain [Narrative Burst → ISR Surge → Sync DDoS → Kinetic Anomaly] within a 24 h window. Grants up to +3 pts bonus if full chain confirmed.
    • 
                
  • HUD — VELOCITY meter, AMBUSH alert, BLOCKADE INDEX gauge, CHAIN badge, Evidence Chain Timeline side panel.

1. Multi-Domain Sensor Architecture (12 Active Sensors)


        
Cyber Domain (Weight: 0.50) — Highest weighted domain. Nation-state cyber operations consistently precede kinetic action by 24–72 hours.

        
    
            
  • Cloudflare Radar API [Poll: 5 min]: L3 (volumetric) and L7 (application-layer) DDoS telemetry. Tracks per-origin traffic percentages globally, computes weighted spike ratios against a rolling 7-day baseline, detects botnet overlap across allied theaters, and identifies L7 vector escalation (HTTP flood shift). This is the highest-signal sensor for detecting coordinated state-directed attack campaigns. Data covers >20% of global internet traffic.
                
    Anti-inflation safeguards: Spike calculation is skipped if the 7-day baseline is not yet populated (startup guard). Spike ratio is capped at 25× to prevent statistical outliers from triggering false alerts. Origins with <1% absolute traffic share are excluded from the weighted average. Non-adversary states use a 3% baseline floor; adversary states (CN/RU/KP) use a 0.5% floor to preserve sensitivity to low-volume state operations.
            
    • 
            
  • ThreatFox (abuse.ch) API [Poll: 30 min]: Queries the ThreatFox threat intelligence feed for APT-tagged IoCs (Indicators of Compromise) published within the past 24 hours. Matches IoCs by APT tag or target country name. A hit indicates that known advanced persistent threat infrastructure is active and associated with the monitored theater. Authentication via THREATFOX_API_KEY (Auth-Key header, required since 2024 policy change).
    • 
            
  • RIPE Stat BGP Routing [Poll: 10 min]: Monitors BGP prefix announcements for the core theater via RIPE NCC. Triggers when announced prefix count drops by >15% relative to the prior poll, indicating mass route withdrawal — a known precursor to internet isolation maneuvers (ref: Russia 2022, Iran 2019).
    • 
            
  • [Phase 8] DDoS Acceleration Engine [Computed in-engine, no external poll]: Computes 1st derivative (velocity, pt/s) and 2nd derivative (acceleration, pt/s²) of the core theater's threat score time series using linear regression over a rolling window (DERIVATIVE_WINDOW=5 points). An Ambush Pattern is declared when the acceleration's Z-Score (vs. its own rolling history) exceeds AMBUSH_ZSCORE_THRESHOLD=2.0 and both velocity and acceleration are positive — indicating exponential escalation rather than linear growth. Adds +2 to Cyber domain when fired. The VELOCITY HUD meter displays escalation rate in threat-points per poll cycle. Destroys Normalcy Bias by showing the direction and rate of change, not just the current level.
    • 
        


        
Physical/Logistics Domain (Weight: 0.30) — Encompasses infrastructure, airspace, maritime, and kinetic pre-strike indicators.

        
    
            
  • IODA API (CAIDA) [Poll: 10 min]: Internet Outage Detection and Analysis. Tracks BGP-based internet reachability and active probing (ping) success rates for each monitored country. A simultaneous drop in both BGP routes and active probing indicates a significant infrastructure outage. This sensor correlates with BGP Routing for confirmation.
    • 
            
  • NASA EONET Wildfire API [Poll: 15 min]: Real-time open wildfire/thermal event data from NASA's Earth Observatory Natural Event Tracker. For each monitored theater, events within ±10° latitude/longitude of the theater's centroid coordinates are flagged. Wildfires near strategic military positions or logistics corridors may indicate kinetic strikes, artillery fires, or pre-conflict infrastructure destruction — particularly when correlated with GDELT tone collapse and cyber spikes. Note: This sensor replaced the original NASA FIRMS endpoint, which is unreachable from this network environment. EONET requires no API key. Coordinates are in GeoJSON [longitude, latitude] format.
    • 
            
  • OpenSky Network (Civilian) [Poll: 3 min]: Civilian airspace monitoring over predefined airport bounding boxes for each theater. Computes a 20-cycle rolling baseline of observed aircraft count. An ANOMALY is triggered when current count deviates by >40% from baseline; CLOSURE is triggered when count drops below 5% of baseline, indicating near-total civilian airspace suspension — historically observed 12–48 hours before military action begins. Weather suppression is applied when OpenWeatherMap reports severe conditions over the target.
    • 
            
  • PeeringDB / IXP Chokepoints [Poll: 60 min]: Static reference layer showing registered Internet Exchange Points (IXPs) and undersea cable landing stations critical to the monitored theater. Used as geographic context for interpreting BGP anomalies and physical infrastructure targeting. Displayed as cyan diamond markers on the map.
    • 
            
  • [Phase 8] ISR Hotspot Sensor (OpenSky — Military ISR) [Poll: 5 min]: Monitors 10 strategically defined ISR_HOTSPOT bounding boxes (200 km radius around Taiwan Strait, East China Sea, Spratlys, Korean DMZ, Black Sea, Baltic, etc.) for Intelligence, Surveillance, and Reconnaissance (ISR) aircraft. Detection logic: aircraft are flagged as ISR-pattern if they meet ANY of: (1) altitude >9,000 m AND speed <160 m/s (slow, high-altitude orbit); (2) squawk code 7777 (government/military transponder); (3) callsign prefix matches known ISR designations (FORTE, JAKE, MYSTIC, RICO, TROLL, etc.). An ISR Surge is declared when ≥3 qualifying aircraft (ISR_SURGE_THRESHOLD) are observed simultaneously within a hotspot. Adds +2 to Physical domain when fired. This sensor is independent of the civilian OpenSky sensor — it monitors sea straits and patrol corridors, not airports.
    • 
            
  • [Phase 8] AIS Maritime Sensor (AISHub) [Poll: 30 min]: Monitors vessels near critical undersea cable chokepoints (Taiwan Strait, Luzon Strait, Miyako Strait, Hormuz, Bab-el-Mandeb, Baltic, Bosphorus, etc.) using AISHub's public position data. Two anomaly types are detected: (1) AIS Dark Gap — a vessel previously tracked near a chokepoint has not transmitted AIS for >AIS_DARK_GAP_THRESHOLD=3600 seconds (1 hour), consistent with deliberate radio silence (e.g., submarine positioning, covert survey vessel); (2) Stationary Anomaly — a non-commercial, non-fishing vessel (AIS ship type not in classes 60–89) is moving at <0.5 knots within AIS_ANCHOR_RADIUS_KM=50 km of a chokepoint. Adds +1 to Physical domain. Dark gap markers are displayed on the map (blue vessel icons).
    • 
        


        
Information Domain (Weight: 0.20) — Propaganda and diplomatic tone analysis. Typically the earliest leading indicator, shifting 48–96 hours ahead of operations.

        
    
            
  • GDELT TimelineTone API [Poll: 15 min]: Queries the Global Database of Events, Language, and Tone for media coverage of the core theater. Computes a rolling 28-day average tone and compares against the configured alert threshold. A tone score below –12.0 constitutes an alert. Calibration reference: Russia→Ukraine onset showed –12 to –15; invasion-day readings reached –18 to –25. The –12.0 threshold provides early-warning sensitivity without excessive peacetime noise. Weather suppression is applied during severe weather conditions.
    • 
            
  • [Phase 8] RSS Narrative Burst Sensor [Poll: 30 min]: Scrapes five state-affiliated RSS feeds simultaneously: TASS (RU), Xinhua (CN), Mehr News (IR), RT, and KCNA Watch (KP). For each theater, a list of theater-specific tactical keywords is matched against article titles and descriptions (e.g., for Taiwan: "military measures," "final warning," "crossing median line," "live-fire exercise"). Daily noise filtering: instead of counting raw hits, the system computes a normalized frequency (keyword hits ÷ total articles) and runs Z-Score analysis against a 30-day rolling baseline. Alert thresholds: Z > NARRATIVE_ZSCORE_ALERT=2.0 → BURST (+1 pt, Info domain); Z > NARRATIVE_ZSCORE_CRITICAL=3.0 → CRITICAL_BURST (+2 pt). Duplicate articles (title similarity >85%) are excluded using difflib. A Z-Score of 2.0 means the current tactical keyword density is 2 standard deviations above the 30-day average — statistically unusual even for adversary state media. The baseline must accumulate ≥7 days of data before alerting begins (cold-start guard). This sensor deliberately ignores steady-state propaganda; it is triggered only by acute, statistically anomalous surges in military-operational language that correlates with actual preparation cycles.
    • 
        


        
Noise Mitigation & Human-in-the-Loop (HITL) Control:

        
    
            
  • OpenWeatherMap [Poll: 30 min]: If severe weather (thunderstorms, blizzard, hurricane, etc.) is detected over the core theater, physical domain alerts (IODA BGP outages, OpenSky airspace anomalies) and information domain alerts (GDELT tone) are automatically marked SUPPRESSED and excluded from scoring. This prevents weather events from creating false escalations.
    • 
            
  • Analyst MUTE (HITL Override): Every sensor entry in the Evidence Panel shows a [ MUTE ] button. Clicking it immediately excludes that sensor's score contribution from the threat calculation and re-runs convergence. Muted sensors are persisted in browser local storage across sessions. Use this to suppress known false positives while preserving the full audit trail. When multiple sensors are muted simultaneously, the system correctly recalculates domain weights with remaining active sensors only. All 12 sensors including Phase 8 additions support MUTE.

3. Scoring Engine & Convergence Algorithm (Phase 8 Extended)

        
The WeightedConvergenceEngine evaluates a Raw Threat Score from all enabled, non-suppressed sensor signals, then applies a cross-domain convergence bonus and a Sequence Bonus. Maximum theoretical score exceeds 22 points (Phase 8).


        
Raw Score Components:

        
    
            
  • [Cyber]    cf_spike_core:       Core theater weighted avg spike >2× (+1), >4× (+1), >6× (+1) — max +3
    • 
            
  • [Cyber]    cf_adversary_strike: Adversary state (CN/RU/KP) direct strike ≥4× spike with >3% abs traffic → +2
    • 
            
  • [Cyber]    cf_botnet_overlap:   Shared botnet overlap with allied theater >30% → +1
    • 
            
  • [Cyber]    cf_vector_shift:     L7 app-layer avg spike >2.5× AND >1.5× greater than L3 avg → +1
    • 
            
  • [Cyber]    cf_coordinated:      ≥2 strategic theaters simultaneously above 3× spike → +1
    • 
            
  • [Cyber]    ripe_bgp:            BGP prefix withdrawal >15% → +1
    • 
            
  • [Cyber]    threatfox:           APT-tagged IoC matched to theater in past 24h → +1
    • 
            
  • [Cyber]    ddos_acceleration:   Ambush Pattern (2nd-deriv Z-Score > 2.0, positive accel) → +2  ★Phase 8
    • 
            
  • [Physical] ioda_bgp:            Confirmed BGP/probing outage (weather-filtered) → +1
    • 
            
  • [Physical] opensky:             Airspace ANOMALY (≥40% deviation from 20-cycle baseline) → +2
    • 
            
  • [Physical] opensky:             Airspace CLOSURE (<5% of baseline, near-total suspension) → +3
    • 
            
  • [Physical] nasa_firms:          NASA EONET wildfire within ±10° of theater centroid → +3
    • 
            
  • [Physical] isr_hotspot:         ISR Surge ≥3 qualifying aircraft at strategic hotspot → +2  ★Phase 8
    • 
            
  • [Physical] ais_maritime:        AIS Dark Gap or Stationary anomaly at chokepoint → +1  ★Phase 8
    • 
            
  • [Info]     gdelt:               GDELT media tone < –12.0 (28-day rolling avg, weather-filtered) → +1
    • 
            
  • [Info]     rss_narrative:       Narrative Burst Z > 2.0 → +1; Critical Burst Z > 3.0 → +2  ★Phase 8
    • 
        


        
Domain Convergence Bonus — Applied when anomalies simultaneously span multiple domains, indicating coordinated multi-domain operations (MDO) rather than isolated incidents:

        
    
            
  • DUAL CONVERGENCE: Any 2 of 3 domains (Cyber + Physical, or Cyber + Info, or Physical + Info) both active → +1 pt
    • 
            
  • FULL CONVERGENCE: All 3 domains (Cyber + Physical + Info) simultaneously active → +2 pts
    • 
        

        
A domain is considered "active" when its weighted score contribution exceeds the domain threshold. Full Convergence is the strongest signal for pre-kinetic operations — historical data shows this pattern in both the 2022 Ukraine invasion and the 2022 Taiwan Strait exercises.


        
[Phase 8] Sequence Bonus — Applied on top of convergence bonus when the logical escalation chain is confirmed within a 24-hour window. This mechanism prevents confirmation bias by requiring cross-domain evidence in the correct logical order of operations:

        
    
            
  • Chain definition (loosely ordered — existence within 24h window, not strict sequential):
                
      [1] NARRATIVE_BURST — Tactical keyword Z-Score spike in state media
                
      [2] ISR_SURGE       — Reconnaissance aircraft detected at strategic hotspot
                
      [3] SYNC_DDOS      — Coordinated DDoS from ≥2 theaters with high botnet overlap
                
      [4] FIRMS_ANOMALY  — Thermal/kinetic event from NASA EONET
    • 
            
  • 4/4 chain confirmed within 24h → SEQUENCE_FULL_BONUS = +3 pts
    • 
            
  • 3/4 chain confirmed within 24h → SEQUENCE_PARTIAL_BONUS = +1 pt
    • 
            
  • <3 events → no bonus (insufficient pattern)
    • 
        

        
Design rationale: The sequence bonus is the system's primary anti-confirmation-bias mechanism. A single dramatic DDoS spike could be a civilian botnet. A narrative burst alone is daily adversary media. Only when multiple independent physical, cyber, and information indicators co-occur in the logical pattern of pre-kinetic preparation does the system grant an escalation bonus — forcing the analyst to see evidence across domains, not just within one.


        
Blockade Index — A computed gauge (displayed in the HUD) that distinguishes "political noise" DDoS from real infrastructure neutralization:

        
    
            
  • Formula:  Blockade Index = min(DDoS_spike_core, 10) ÷ (0.1 if BGP_degraded else 1.0)
    • 
            
  • NORMAL               — Index < 1.5: DDoS present but infrastructure remains reachable. Typical of signaling operations.
    • 
            
  • POLITICAL_NOISE      — Index 1.5–3.9: Significant cyber activity, no confirmed infrastructure impact.
    • 
            
  • SEVERE_DISRUPTION    — Index 4.0–6.9: High DDoS concurrent with partial BGP degradation.
    • 
            
  • INFRASTRUCTURE_NEUTRALIZATION — Index ≥ 7.0: Massive DDoS + confirmed BGP outage. Pre-invasion blackout signature.
    • 
        


        
Botnet Overlap Calibration: The 30% threshold reflects confirmed nation-state coordinated attack patterns (20–35% shared botnet infrastructure, per post-incident forensics of GRU/Sandworm operations). The previous 45% threshold matched large civilian botnet patterns (Mirai-class) and was too insensitive to state-directed attacks that use diverse, smaller botnets.


        
4. DEFCON Threat Levels

        
Final Score = Raw Score + Convergence Bonus. Hysteresis (1-cycle delay) is applied when downgrading levels to prevent flickering on borderline readings. Upgrades are immediate.

        
    
            
  • Level 1 — CRITICAL: Score ≥ 9 AND Hard Condition (Core Spike >5.0× AND confirmed BGP outage simultaneously). This compound condition requires both massive cyber surge and infrastructure collapse — the signature of a coordinated pre-invasion blackout operation. Imminent kinetic action probability is extremely high. Activate all contingency procedures.
    • 
            
  • Level 2 — SEVERE: Score ≥ 6. Multi-domain threat with direct adversary state attribution or severe infrastructure degradation. High probability of state-directed hostile action. Immediate command notification required.
    • 
            
  • Level 3 — HIGH: Score ≥ 4. Significant coordinated anomaly across at least two sensors. May indicate preparation phase, probing operations, or early escalation. Heightened monitoring warranted.
    • 
            
  • Level 4 — ELEVATED: Score ≥ 2. Early-stage indicators: media tone deterioration, isolated BGP events, or minor L7 vector shifts. Consistent with pre-crisis diplomatic deterioration or reconnaissance activity.
    • 
            
  • Level 5 — NORMAL: Score < 2. Background noise level. No significant anomalies across monitored domains.
    • 
        


        
5. Historical Calibration Reference

        
Thresholds in this system are calibrated against the following confirmed events:

        
    
            
  • Russia → Ukraine (Feb 24, 2022): BGP route withdrawals 10–15% in the 6 hours before invasion. Wiper malware (HermeticWiper) deployed via pre-positioned APT infrastructure 90 min before ground assault. GDELT tone reached –18 to –25 at invasion onset; –12 to –15 during the preceding weeks of military buildup. Civilian flights suspended 24h prior.
    • 
            
  • China → Taiwan Strait (Aug 2022, Pelosi visit): CN-origin DDoS traffic spiked 3–5× against TW government and media infrastructure. GDELT tone –13 to –17 during the exercise period. No BGP outage (deliberate — PRC signaling, not isolation). Botnet overlap with JP/US-targeted traffic confirmed in post-incident analysis.
    • 
            
  • DPRK persistent operations (ongoing): KP-origin traffic characteristically low in absolute percentage (0.1–0.5%) but highly persistent and targeted. The 0.5% adversary baseline floor preserves detection of these small-volume, high-significance operations that would be masked by a 3% floor.
    • 
        


        
6. Analyst Workflow & Best Practices

        
    
            
  • Cross-domain confirmation first: Do not escalate based on a single domain. A DDoS spike alone (Cyber only) is insufficient — civilian botnets routinely produce 3–5× spikes. Full Convergence (Cyber + Physical + Info) is the minimum standard for Level 2 escalation recommendations.
    • 
            
  • Use HITL MUTE for known false positives: If a sensor is producing noise from a confirmed non-hostile source (e.g., a major sporting event causing airspace anomalies, or a legitimate CDN maintenance window causing BGP shifts), mute it in the Evidence Panel. The mute state persists across page reloads via browser local storage.
    • 
            
  • SITREP for command reporting: Use the SITREP button to generate a formatted Situation Report suitable for command-level notification. The report includes domain breakdown, sensor evidence, and trend indicators.
    • 
            
  • Baseline awareness: Spike ratios are only computed when the 7-day rolling baseline has been populated. On first startup, the system requires approximately 7 days of sensor operation to establish accurate baselines. During this period, spike scores will be conservatively suppressed (has_baseline guard active). The Data Status panel shows baseline population progress per sensor. [Phase 8] The RSS Narrative Sensor similarly requires ≥7 days of data before its Z-Score alerts are active.
    • 
            
  • GDELT tone interpretation: Tone of –12.0 marks the threshold for early military tension rhetoric in open media. Values of –15 to –18 indicate active crisis framing. Values below –18 historically correspond to imminent or ongoing kinetic action. A sudden 3–5 point tone drop within a single polling cycle (15 min) is more significant than a gradual decline to the same value.
    • 
            
  • [Phase 8] CHAIN panel for escalation narrative: Click the CHAIN button (top-right of HUD) to open the Evidence Chain Timeline. This panel shows the last 24 hours of escalation events (Narrative Burst → ISR Surge → Sync DDoS → Kinetic Anomaly) with timestamps, enabling the analyst to see the "story" behind the threat score — not just a number. A FULL CHAIN confirmation (+3 pts) means all four event types occurred within 24 hours and is a strong indicator of coordinated multi-domain preparation.
    • 
            
  • [Phase 8] VELOCITY meter interpretation: The VELOCITY display shows threat-score change rate in points-per-poll-cycle (≈900 seconds). A stable situation reads "STABLE" (grey). Positive velocity (red, ↑) means the threat level is actively rising. Negative velocity (green, ↓) means the situation is improving. An ⚡ AMBUSH alert (blinking red) means the rate of change itself is accelerating exponentially — the system detected a 2nd-derivative spike, the mathematical signature of a rapidly compressing attack window rather than a gradual escalation. This is the most time-critical signal in the system.
    • 
            
  • [Phase 8] Blockade Index interpretation: The BLOCKADE gauge on the HUD distinguishes DDoS operations by their actual infrastructure impact. A high-volume DDoS that doesn't degrade BGP reachability (Index < 1.5) is political signaling. An INFRASTRUCTURE_NEUTRALIZATION reading (Index ≥ 7.0) means the adversary is achieving actual internet blackout alongside the DDoS — the pattern seen 30–90 minutes before the Russian ground invasion of Ukraine in February 2022.
    •

  • GET /api/historical_events [Intuition UI] — Returns the HISTORICAL_EVENTS pattern library from geo_data.json: 5 reference crisis fingerprints (Ukraine 2022, Taiwan Strait 2022, Iran 2019, Georgia 2008, Normal Baseline), each with a 20-point normalized pattern array, color, and description. Used by the Historical Analog panel.
    • 
        


        
8. Intuition UI — Recognition-Primed Decision (RPD) Components

        
Phase 8 追加機能。数値計算だけでなく、アナリストの経験則・直観（Recognition-Primed Decision）を刺激する8つのビジュアルコンポーネント。NDM（Naturalistic Decision Making）理論に基づき、状況認識を即座に形成することを目的としている。すべて既存の数値表示と共存する。


        

            
操作方法

            
HUD右上の TOOLS ▾ ボタンをクリックするとドロップダウンメニューが開き、Weather Brief / SALUTE Board / Historical Analog / Ops Clock の4パネルを個別に有効化できる。アクティブなパネルは シアン 点灯で示される。各パネル右上の ✕ で個別に閉じることができ、パネルを閉じてもドロップダウンの状態インジケーターが即座に同期される。
常時表示のコンポーネント（Pulse Display / Radio Silence / Ambient Atmosphere / Threat Terrain）はボタン操作不要で自動的に動作する。

        


        
A. Ambient Atmosphere（環境光）

        
    
            
  • 機能: 脅威レベルに応じて画面全体の背景色を極微妙に変色させる。Lv 3(HIGH)→緑み、Lv 2(SEVERE)→赤橙み、Lv 1(CRITICAL)→深紅み、2〜5秒周期のkeyframesアニメーション。
    • 
            
  • 設計意図: 意識的に数値を読まなくても、周辺視野で「何かがおかしい」と気付かせる。長時間監視セッションにおける注意の固着（Attentional Tunneling）を防ぐ。
    • 
            
  • 常時動作: 自動的に適用される。明示的な操作不要。
    • 
        


        
B. Pulse Display（脅威パルス波形）

        
    
            
  • 表示位置: マップ左下コーナー、常時表示。
    • 
            
  • 機能: 脅威レベル履歴（最大60サイクル ≈ 15時間）をスクロール型EKG波形Canvasで描画。DEFCON 1(危険)が波形上端、5(平常)が下端。波形の色は現在のDEFCONカラーと連動する（緑/黄/橙/赤）。右上に変化率（/時間）を表示。
    • 
            
  • 読み方: 波形が上方向に急峻に動いている → 急速に状況が悪化している。平坦な高止まり → 継続的な高脅威環境。下方向に動き始めた → 緩和の兆候。単なる数値より「物語」として認識できる。
    • 
        


        
C. Operational Clock（作戦時計）

        
    
            
  • アクセス: TOOLS ▾ → Ops Clock
    • 
            
  • 機能: リアルタイムのZulu（UTC）時刻を大型フォントで表示（秒単位更新）。ローカル時刻・UTCカレンダー日付・「最後に重大イベント（Lv 3以下）が発生してからの経過時間」を同時表示。
    • 
            
  • 設計意図: 軍事作戦において時刻はすべての報告書の基準である。アナリストが報告書を作成する際、別ウインドウで時計を参照する手間を省く。「最終イベントからの経過時間」は状況が安定しているのか単に沈黙しているのかを区別するための手掛かりになる。
    • 
        


        
D. Operational Weather Brief（作戦気象概況）

        
    
            
  • アクセス: TOOLS ▾ → Weather Brief
    • 
            
  • 機能: GET /api/weather_brief を呼び出し、Cyber/Physical/Info各ドメインの脅威状態を気象メタファーで表示する。例: Cyber=「サイバーストーム(STORM)」= 大規模DDoSスパイク活発。Physical=「海峡霧(FOG)」= AIS暗区間発生中。Info=「気圧上昇(RISING PRESSURE)」= ナラティブ工作が強まっている。
    • 
            
  • 設計意図: 気象予報は専門家でなくても直観的に理解できる唯一の科学的表現系の一つ。「明日は嵐です」が「降水確率87%」より行動につながりやすいのと同じ原理。意思決定速度の向上を目的とする。
    • 
            
  • データソース: /api/weather_brief — センサー値をthreshold比較してcondition_keyを決定し、その結果をフロントエンドがアイコン付きで描画する。
    • 
        


        
E. SALUTE Board（接触報告板）

        
    
            
  • アクセス: TOOLS ▾ → SALUTE Board
    • 
            
  • 機能: GET /api/salute_report から軍事接触報告書（SALUTE様式）を生成する。
    • 
            
  • SALUTE様式:
                
      
                    
    • S ize     — 敵勢力規模 (例: Botnet 12 origins, 3 state-ASN)
      • 
                    
    • A ctivity — 確認された活動種別 (例: Coordinated L7 surge + Narrative Burst)
      • 
                    
    • L ocation — 標的地域 (例: TW, correlated JP/US)
      • 
                    
    • U nit     — 帰属推定 (例: CN/RU attributed — state-ASN matched)
      • 
                    
    • T ime     — 最初の活動観測時刻 (UTC)
      • 
                    
    • E quipment — 使用手段 (例: L3 volumetric + BGP withdrawal + ISR surge)
      • 
                    
    • + Assessment — 統合評価文（AIが現在の脅威状態から自動生成）
      • 
                
    
            
    • 
            
  • 設計意図: ブリーフィング・指揮報告に即座に使える標準化フォーマット。センサーデータを「数値の羅列」から「報告文」へ変換することで、指揮官への伝達速度を向上させる。
    • 
        


        
F. Historical Pattern Analog（歴史的類似事例）

        
    
            
  • アクセス: TOOLS ▾ → Historical Analog
    • 
            
  • 機能: 現在の脅威スコア時系列（最新20サイクル）を、geo_data.json に収録された5件の歴史的事例パターンとPearson相関係数で比較し、類似度の高い順に上位3件を表示する。各カードに類似度%と「過去パターン（dim）vs 現在パターン（cyan）」の重ね合わせグラフを描画。
    • 
            
  • 収録パターン:
                
      
                    
    • 🔴 Ukraine Invasion 2022 — BGP撤退→ワイパー展開→地上侵攻の前日パターン
      • 
                    
    • 🟠 Taiwan Strait 2022 — ペロシ訪問前後: 政治的シグナリングDDoS+GDELT急落
      • 
                    
    • 🟡 Iran Shutdown 2019 — 国内選挙前のインターネット完全遮断パターン
      • 
                    
    • 🔵 Georgia-Russia 2008 — 先行サイバー作戦から地上侵攻までの上昇曲線
      • 
                    
    • ⬜ Normal Baseline — 平常時ノイズ（参照基準）
      • 
                
    
            
    • 
            
  • 設計意図: 「現在の状況はどの過去事例に似ているか」を直観的に示す。Pearson相関で70%以上の類似度 = 統計的に注目に値するパターン一致。分析官はこれを出発点として歴史的文脈と現在の差異を検討する。相関 ≠ 因果: これは意思決定補助であり予測ではない。
    • 
        


        
G. Radio Silence Indicator（無線通信途絶インジケーター）

        
    
            
  • 表示位置: HUD下段、常時表示。🟢 LIVE / 🟠 QUIET の2状態。
    • 
            
  • 機能: 「脅威スコアが有意（≥3点）であるにもかかわらず、速度（velocity）変化がほぼゼロ」の状態を検出し QUIET と判定する。
    • 
            
  • 何を示すか:
                
      
                    
    • LIVE: 全センサーが変動を報告中。通常の監視状態。
      • 
                    
    • QUIET: 異常な静寂。以下の可能性を示唆:
                        (1) 作戦前の通信封鎖（pre-kinetic EMCON）、
                        (2) センサー抑圧（adversary jamming/spoofing）、
                        (3) 攻撃者が戦術的一時停止に入っている。
                    
      • 
                
    
            
    • 
            
  • 設計意図: 真の静寂は平和を意味しない。ロシア→ウクライナ作戦では、侵攻直前の数時間に一部センサーが「静止」するパターンが確認されている。「何も起きていない」という誤認を防ぐため、静寂そのものを異常検出の対象とする。
    • 
        


        
H. Threat Terrain Overlay（脅威地形オーバーレイ）

        
    
            
  • アクセス: マップ右上のレイヤーコントロール → Threat Terrain にチェックを入れる。
    • 
            
  • 機能: 監視中の各国のDDoSスパイク強度（avg_spike）に応じて、Leafletの半透明サークルをchoropleth風に描画する。BGP障害が確認された国は赤色、スパイク4倍以上はオレンジ、それ以下は黄色。半径はスパイク強度に比例（最大約620km）。
    • 
            
  • 設計意図: 個別の数値ではなく「地理的な圧力分布」として脅威を認識させる。どの地域に負荷が集中しているかを一瞥で把握でき、従来の折れ線グラフでは見えにくい「地政学的パターン」を浮き上がらせる。スパイク平常以下の国はオーバーレイを描画しないためノイズを最小化する。
    •
